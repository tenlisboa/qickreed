| AD-006 | **Training surface = scoped dark palette** (`.training-surface` class on the RSVP tachistoscope container only). PRD Phase 3 mandates a dark background for the digital tachistoscope; this does **not** change the app-wide light theme. Tokens `--training-bg` (`#000000`), `--training-fg` (`#ffffff`), `--training-border` (`#ffffff`) live in `:root` next to the neobrutalism tokens; the `.training-surface` class applies them locally. `--main` (`#FFD23F`) is reused for the progress fill and break-screen accents so the two-accent rule is preserved. The dark surface is reachable only via the `(immersive)` route group (no Sidebar), keeping the rest of the app neobrutalist light. | active | 2026-07-16 | QICA-13 RSVP immersion — needed a dark tachistoscope without violating AD-005; scoped class instead of swapping `:root` keeps the rest of the app light. |
| AD-007 | **Assessment baseline timer starts on user intent (click-to-reveal), not on page load.** The reading page fetches the diagnostic text asynchronously (`getTextById`), so a literal "starts on page load" timer would bill network/render latency against the user's baseline WPM — systematically biasing slow connections and weak devices downward. The timer is invisible (no `MM:SS.t` face) **and** its start signal is the "Começar a Ler" click, which reveals the text and starts timing in the same tick. MVP_SCOPE.md §2 Phase 1 amended accordingly. | active | 2026-07-16 | task-001 — user raised the connection/device bias; start signal = user intent guarantees text is rendered before counting starts. |
